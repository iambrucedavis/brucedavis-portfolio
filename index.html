I'm a designer who owns the build. I draft static surfaces in Figma, interactive ones in code — wherever each is most legible. From first sketch to live URL, same hand.

I built systems before I designed them.

CS degree. Master's in computer graphics technology. Fifteen years in tech. The work that felt alive was always the work where decisions were still visible — the edit legible, the structure exposed.

I work in Figma when the surface is static. I work in code when the surface moves. The cursor effect, the streaming UI, the loading state — these are decisions, not decorations. They are most legible in the medium where they will run.

First sketch to live URL. One hand the whole way.

How It Works

Three moves.
One ship.

  1. // start here
    01
    Bring The Idea
    A product, an interaction, a feature, a brief. We talk it through. I sketch in Figma where it helps, in code where it helps more. The output is a prototype, not a deck.
  2. // own it
    02
    Build The Real Thing
    Production code from day one. The interaction is built in build. The animation runs in the browser. The state and motion are owned by the same hand. No handoff between design and engineering — same person.
  3. // live
    03
    Push To Live
    Deploy. Iterate. Polish in public. Micro-interactions tightened, motion timing refined, accessibility checked. The work is the URL.
